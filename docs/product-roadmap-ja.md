# Day Manager 商品レベル化ロードマップ（無料前提）

## 0. 前提

このアプリは**大学生向けの「その日の再設計」特化ツール**として磨く。

目標は以下。

- 毎朝 30 秒〜2 分でその日の動き方を決められる
- AIがなくても成立する
- 完全無料で運用できる
- 一般商品レベルの見た目・操作性を目指す
- ただし利用者は当面は自分1人でよい

### 採用する前提
- フロント主体
- PWA 継続
- localStorage + JSONバックアップ継続
- 必要なら個人無料枠のある外部サービスは使う

### 直近ではやらないこと
- 複雑な双方向同期
- 本格的な年次計画UI
- Drag & Drop
- AIがないと成立しない設計
- 一般向けの公開配布前提の重い運用基盤

---

## 1. 商品コンセプト

### 一言で言うと
> 授業・課題・予定が混ざる大学生の1日を、毎朝すばやく組み直すアプリ

### 主価値
- 今日やるべきことがすぐ見える
- 崩れた日でも立て直せる
- 予定・タスク・体調を分けずに扱える
- AIは補助として使える

## 2. いままでの改善点の整理

## 採用済み
- 画面の大分割（今日の操作と管理領域の分離）
- 前日 / 今日 / 翌日ナビ
- 縦長フォームの折りたたみ
- Gemini提案JSONの読込と反映の土台
- 長期計画用 state / 日付ユーティリティの準備
- 条件入力のデバウンス
- バックアップURL revoke の遅延

## 修正不足 / 再修正対象
- タスク / 単発予定 / 固定予定の**一覧本体**は主画面に置く
- フォームだけ管理領域に残す
- 要約カードが多すぎるので削減する
- 一覧の視認性が低い（文字列詰め込み）
- alert / confirm がまだ残っている
- Gemini UI が動的注入のまま

## 明確に後回しにするもの
- Drag & Drop
- 完全自動AIスケジューリング
- 一般向けGoogle Calendarフル同期
- 年間計画の本格UI

---

## 3. 無料前提の技術方針

## 必須コア
- GitHub Pages か静的配信
- localStorage 保存
- JSON バックアップ
- PWA

## 使ってよい外部サービス（個人無料前提）
- Firebase Authentication（Googleログイン）
- Firestore（必要になったら最小限）
- Gemini は当面コピペ連携

## 当面採用しないもの
- OpenAI API
- サーバー必須構成
- 課金前提のAI API運用
- Firebase Functions 前提の構成

## 4. 実装ロードマップ

## Phase 1: 商品らしい主画面にする

### ゴール
今日の情報を見て、そのまま行動に移せる画面にする。

### タスク
1. タスク / 単発予定 / 固定予定の一覧を主画面へ戻す
2. 要約カードを整理する
   - 「今日の状態」と「今日の見取り図」を統合寄りにする
   - 重複した要約は削る
3. クイック追加の結果が見える位置に一覧を置く
4. 編集時は管理パネルが自動で開く

### 完了条件
- 主画面だけで「今日の確認」と「小さな更新」ができる
- 管理パネルを開かなくても日常運用が回る

---

## Phase 2: リストを商品レベルの見た目にする

### ゴール
文字の羅列ではなく、見た瞬間に重要度が分かる一覧にする。

### タスク
1. タスク一覧のメタ文字列を分解する
2. バッジを追加する
   - 優先度
   - 重要度
   - 状態
3. 締切の近いタスクを強調する
4. 完了タスクを薄くする
5. 単発予定と固定予定もステータス表示を整理する

### 完了条件
- スラッシュ区切りの文字列を読まなくても状態が分かる
- 完了済みと未完了が一目で区別できる

---

## Phase 3: ネイティブダイアログを廃止する

### ゴール
alert / confirm なしで完結する操作体験にする。

### タスク
1. トースト通知を追加
2. インラインエラー表示を追加
3. 削除確認モーダルを追加
4. Undo削除を追加

### 完了条件
- alert / confirm がコードからなくなる
- 主要操作がすべてアプリ内UIで完結する

---

## Phase 4: 入力負荷を下げる

### ゴール
アプリがAIのための入力フォームではなく、人間のための時短ツールになる。

### タスク
1. クイック追加の正規化強化
   - 全角半角
   - 1h / 30m
   - 1限 / 2限
   - 今日中 / 今夜
2. よく使うテンプレート追加
   - 授業
   - レポート
   - 復習
   - 外出
3. 昨日の未完了を持ち越し候補として表示
4. 体力 / 睡眠入力を1タップ化

### 完了条件
- 追加フォームを開く頻度がかなり減る
- 1行入力だけでも実用になる

---

## Phase 5: AIを補助機能として仕上げる

### ゴール
AIがなくても成立し、AIがあると再設計が速い状態にする。

### タスク
1. Gemini セクションを静的HTML化
2. AI提案一覧の衝突表示
3. 重複候補の明示
4. 週プロンプト生成
5. AI提案をローカルに入れる導線を改善

### 完了条件
- Geminiの利用が迷わない
- JSON貼り付けでも十分使える
- AIを使わない日でも不便ではない

---

## Phase 6: 個人無料サービスを使った拡張

### ゴール
自分だけで使う商品レベルとして、ログインと端末横断の基盤を整える。

### タスク
1. Firebase Authentication で Googleログインを追加
2. ログイン状態表示を追加
3. local モード / login モードを分ける
4. 必要なら Firestore で設定とデータを同期
5. Google Calendar β は奥に置いたまま維持

### 完了条件
- ログイン導線がある
- 端末をまたぐ土台ができる
- ただしローカル単独でも成立する

## 7. 受け入れ基準

以下を満たしたら「商品レベルに近づいた」と判断する。

- 初回1分以内に何をすればいいか分かる
- 今日のタスクと予定が最初に見える
- クイック追加後、反映結果がすぐ分かる
- 完了済み / 危険 / 必須 が一目で分かる
- alert / confirm がない
- AIを使わなくても日常運用できる
- AIを使う場合も迷わない

---

## 8. 次の具体タスク

次の実装はこの2本に限定する。

1. タスク / 単発予定 / 固定予定の一覧を主画面へ戻す
2. render.js / style.css を改修して、タスク一覧をバッジ式UIにする

この2つが終わってから、alert / confirm 廃止に入る。
